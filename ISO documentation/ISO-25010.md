# ISO/IEC 25010 — Software Quality Model

El modelo de calidad del producto software se compone de 8 características principales. A continuación se mapea cada una a implementaciones concretas en MoneyMe.

## 1. Functional Suitability (Funcionalidad)

| Sub-característica | Implementación en MoneyMe |
|---|---|
| **Functional completeness** | Backend con 11 módulos API: auth, transactions, wallets, categories, dashboard, budgets, OCR, analysis, predictions, reports, export |
| **Functional correctness** | 68+ tests unitarios backend cubriendo casos de éxito, error y borde. Schemas Pydantic validan datos en entrada |
| **Functional appropriateness** | Frontend Flutter con Clean Architecture: data/domain/presentation separados. Provider para estado. UI responsive (mobile/tablet/desktop) |

## 2. Performance Efficiency (Rendimiento)

| Sub-característica | Implementación |
|---|---|
| **Time behaviour** | Consultas asíncronas SQLAlchemy. Paginación en listados (`limit`/`offset`). Lazy loading en gráficas CustomPaint |
| **Resource utilization** | OCR cleanup service programado cada 24h. Conexiones DB con pool (`pool_pre_ping`, `pool_size`) |
| **Capacity** | MySQL local escalable. SQLite en tests para aislamiento rápido |

## 3. Compatibility (Compatibilidad)

| Sub-característica | Implementación |
|---|---|
| **Co-existence** | API versionada (`/api/v1/`). CORS configurado para múltiples orígenes locales |
| **Interoperability** | API REST + JSON. Frontend Flutter web (Chrome). Backend independiente del frontend |

## 4. Usability (Usabilidad)

| Sub-característica | Implementación |
|---|---|
| **Appropriateness recognizability** | Prototipos en `docs/prototype-map.md` con mapa de navegación y matriz de estados |
| **Learnability** | Login/Register en mismo widget (toggle in-place). CTAs visibles en FABs consistentes |
| **Operability** | SafeArea, padding consistente, radius máximo 14px, cards con border en vez de shadow |
| **User error protection** | Validación de cantidad negativa, categoría duplicada, wallet inexistente, contraseña débil |
| **Accessibility** | Paleta corporativa de alto contraste (navy + muted blue). withValues(alpha:) para opacidad |

## 5. Reliability (Confiabilidad)

| Sub-característica | Implementación |
|---|---|
| **Maturity** | Manejo global de excepciones (`global_exception_handler`, `http_exception_handler`). Errores 422 para validación, 401/403 para auth |
| **Fault tolerance** | API stateless, JWT auto-contenido. Refresh token permite sesión continua |
| **Recoverability** | DB transaccional (commit/rollback). Cleanup service automático para capturas OCR huérfanas |

## 6. Security (Seguridad)

| Sub-característica | Implementación |
|---|---|
| **Confidentiality** | Passwords hasheados con `hash_password`. JWT con expiración. Variables sensibles en `.env` |
| **Integrity** | Datos en cents (enteros) para evitar errores de decimales. Transacciones atómicas SQLAlchemy |
| **Non-repudiation** | Logs SQLAlchemy habilitados. Trazabilidad de operaciones |
| **Accountability** | `get_current_user()` con JWT. Cada recurso filtrado por `user_id` |
| **Authenticity** | Header Authorization con Bearer JWT. Test de aislamiento (User A no ve datos de User B) |

## 7. Maintainability (Mantenibilidad)

| Sub-característica | Implementación |
|---|---|
| **Modularity** | Feature-based modules en backend y frontend. Clean Architecture en Flutter |
| **Reusability** | Use cases reutilizables. Schemas Pydantic compartidos. Widgets atómicos (MoneyCard, MoneyButton, MoneyFormField) |
| **Analysability** | Logs estructurados. Tipado estricto Python + Dart. Tests de regresión |
| **Modifiability** | API versionada permite cambios sin romper clientes. .env para configuración sin tocar código |
| **Testability** | pytest con SQLite en memoria. Fixtures reutilizables. 68+ tests backend + 14 frontend |

## 8. Portability (Portabilidad)

| Sub-característica | Implementación |
|---|---|
| **Adaptability** | Backend portable: PostgreSQL, MySQL o SQLite según `DATABASE_URL`. `normalize_database_url()` convierte drivers |
| **Installability** | `setup_local.ps1` + `run_local.ps1` para Windows. XAMPP para MySQL local |
| **Replaceability**| Tesseract OCR (spa+eng) configurable por env. reportlab para PDF sin wkhtmltopdf |
