# ISO/IEC 12207 — Software Life Cycle Processes

## Procesos aplicados en MoneyMe

### 1. Procesos de Acuerdo (Agreement Processes)

| Proceso | Aplicación |
|---|---|
| **Adquisición** | Definición de requisitos en `Planificaciones/`. Prototipos en `docs/prototype-map.md` |
| **Suministro** | API REST documentada en `/docs` (Swagger/OpenAPI). Frontend Flutter web como producto entregable |

### 2. Procesos Organizacionales (Organizational Project-Enabling Processes)

| Proceso | Aplicación |
|---|---|
| **Gestión del ciclo de vida** | Fases 1-12 definidas y ejecutadas secuencialmente. Siguiente: Fase 13 (E2E), 14 (performance) |
| **Gestión de infraestructura** | Backend: Python FastAPI + SQLAlchemy + MySQL. Frontend: Flutter web. Tests: pytest. CI/CD planeado |
| **Gestión de recursos humanos** | Proyecto unipersonal, documentado para transferencia. AGENTS.md con instrucciones para asistentes IA |
| **Gestión de calidad** | Tests unitarios (68+ backend, 14 frontend). Plan de pruebas en `Planificaciones/` (11 documentos) |

### 3. Procesos Técnicos (Technical Processes)

| Proceso | Aplicación |
|---|---|
| **Definición de requisitos** | `docs/requirements.md` (si existe). Prototipos y matriz de estados en `prototype-map.md` |
| **Análisis de requisitos** | Desglose en issues/features. Validación mediante tests de aceptación |
| **Diseño arquitectónico** | Clean Architecture (domain/application/infrastructure/api). Feature-based modules. API versionada |
| **Implementación** | Código en `src/` (backend) y `lib/` (frontend). Tipado estricto. Sin comentarios redundantes |
| **Integración** | Backend + Frontend integrados via API REST. Tests de integración planeados (Fase 13) |
| **Verificación** | 68 tests backend + 14 frontend. pytest con SQLite en memoria para aislamiento |
| **Validación** | Prototipos funcionales. Pruebas de aceptación documentadas en `Planificaciones/08-aceptación.md` |
| **Mantenimiento** | `cleanup_loop` para OCR. Actualizaciones de dependencias. Migración de PostgreSQL → MySQL |

### 4. Procesos de Soporte (Support Processes)

| Proceso | Aplicación |
|---|---|
| **Documentación** | `docs/`, `Planificaciones/`, `Estados del Arte/`, `Diseño/`, `ISO documentation/` |
| **Gestión de configuración** | Cambios rastreados manualmente. `.env` para configuración. Dependencias en `requirements.txt` y `pubspec.yaml` |
| **Aseguramiento de calidad** | Pre-commit checks (lint + typecheck). FastAPIDeprecationWarning monitoreado |
| **Verificación** | pytest para backend. Flutter test para frontend. Pruebas de regresión en suite completa |
| **Revisión** | Código revisado contra convenciones: radius 14px máximo, sin pill shapes, withValues() en vez de withOpacity() |
| **Auditoría** | Logs SQLAlchemy. Trazabilidad de errores via `error_handlers.py` |

## Ciclo de Vida iterativo-incremental

```
Fase 1-6: Core (auth, transactions, wallets, categories, dashboard)
Fase 7-8: OCR + Reports/Export
Fase 9-12: Analysis, Predictions, Prototipos, UI refinamiento
Fase 13:   Tests E2E
Fase 14:   Performance optimization
```
