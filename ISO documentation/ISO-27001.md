## A.9 — Control de Acceso

| Control | Implementación |
|---|---|
| **A.9.1.2** Acceso a redes y servicios | CORS limitado a `localhost:3000,5000,8080,127.0.0.1:8080`. API detrás de puerto local |
| **A.9.2.1** Registro y baja de usuarios | `POST /auth/register` con validación de email y contraseña. `DELETE /auth/account` para baja |
| **A.9.2.2** Acceso privilegiado | `get_current_user()` extrae `sub` del JWT. Cada endpoint verifica propiedad del recurso por `user_id` |
| **A.9.4.2** Autenticación segura | Passwords hasheados con `hash_password`. JWT firmado con algoritmo HS256. Header `Authorization: Bearer` |

### Flujo de autenticación

```
Cliente → POST /auth/login → email + password
  → Verifica hash → genera JWT (sub=user_id, email, exp)
  → Cliente almacena token → lo envía en cada request como Bearer
  → GET /auth/me valida token → devuelve perfil + wallets
```

### Pruebas de seguridad implementadas

- `test_get_me_expired_token` — token vencido rechazado
- `test_get_me_malformed_token` — token inválido rechazado
- `test_get_me_wrong_scheme` — esquema Basic rechazado
- `test_transaction_isolation` — User A no accede datos de User B
- `test_categories_unauthorized` — sin token devuelve 401
